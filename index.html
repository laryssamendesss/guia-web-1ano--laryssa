1. O que são HTML, CSS e JavaScript?

HTML — Estrutura

HTML (HyperText Markup Language) é a linguagem de marcação que define a estrutura do conteúdo na web: títulos, parágrafos, listas, links, formulários e mais. Elementos semânticos ajudam navegadores e tecnologias assistivas a entender o conteúdo.

CSS — Apresentação

CSS (Cascading Style Sheets) controla a aparência: cores, tipografia, layout e responsividade. Separar estilo da estrutura permite manutenção e temas.

JavaScript — Comportamento

JavaScript adiciona interatividade e dinâmica: manipula o DOM, trata eventos, valida formulários, faz requisições HTTP (fetch) e interage com APIs do browser (localStorage, Geolocation, Service Workers, etc.).

2. Boas práticas profissionais

Acessibilidade (a11y)

  • Use marcação semântica (<nav>, <main>, <header>, <footer>, etc.).
  • Forneça textos alternativos (alt) e rótulos (labels) claros.
  • Controle o foco, use contrastes adequados e teste com teclado e leitores de tela.

Semântica

Marcação semântica melhora SEO e manutenção. Escolha a tag correta (por exemplo, <button> em vez de <div> para ações).

Responsividade

Adote design fluido, media queries e unidades relativas (rem, em, %, vw/vh) para suportar diversos tamanhos de tela.

Organização

  • Mantenha arquivos pequenos e modulares (CSS e JS separados).
  • Comente e documente decisões importantes.

Versionamento

Use Git: commits pequenos, mensagens descritivas, branches para features e tags para releases.

3. Panorama de tecnologias web

Front-end

HTML/CSS/JS no browser. Ferramentas comuns: bundlers, preprocessadores, transpiladores (nem sempre necessários quando se usa apenas ES6+).

Back-end (visão geral)

Servidores que processam lógica, autenticação e armazenam dados. Linguagens comuns: Node.js, Python, Ruby, Java, PHP. Neste projeto não usamos back-end.

Bancos de dados

Relacionais (Postgres, MySQL) ou NoSQL (MongoDB). No browser, para persistência simples, use localStorage, IndexedDB ou APIs remotas.

DevOps

Build, CI/CD, containers, e serviços de hosting (Netlify, GitHub Pages, Vercel). Automatize testes e deploys para consistência.

Testes

Testes unitários, de integração e end-to-end. Para sites estáticos, inclua testes de acessibilidade e regressão visual quando possível.

Como escolher?

  1. Defina requisitos (escala, tempo, orçamento).
  2. Escolha ferramentas que o time domina.
  3. Prefira soluções simples para MVP; otimize depois.
